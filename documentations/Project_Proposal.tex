\section{Project Overview}

\subsection{Origin and Inspiration}

The concept of Grid Survival emerged during our Game Development course at ICT University, where students were tasked with creating a fully functional, playable game with multiplayer support. Our team drew inspiration from the competitive minigame format popularized by Mario Party and similar party games, which combine simple controls, quick match duration (2–5 minutes), and intense competition.

We identified a gap in accessible, LAN-based competitive games suitable for classroom and campus settings, and designed Grid Survival to fill that niche: a game that is quick to learn, fun to play, and technically sophisticated enough to serve as a portfolio piece demonstrating modern game development practices.

\subsection{Game Concept}

\textbf{Title:} Grid Survival \\
\textbf{Genre:} Multiplayer Party Game \\
\textbf{Platform:} PC (Windows 10/11) \\
\textbf{Player Count:} 3–4 players per match \\
\textbf{Match Duration:} 5–15 minutes \\
\textbf{Core Mechanics:}
\begin{itemize}
    \item Navigate a disappearing tile grid while avoiding environmental hazards
    \item Collect power-up orbs that grant temporary abilities
    \item Outlast opponents through skill, timing, and strategy
    \item Compete in Campaign, LAN, and Online modes
\end{itemize}

\section{Game Modes}

Grid Survival supports four distinct gameplay modes, each optimized for different player counts and network environments:

\subsection{Campaign Mode (Single-Player vs. AI)}

Campaign Mode allows a single player to compete against AI-controlled opponents in a series of matches. This mode is ideal for:
\begin{itemize}
    \item Learning the game mechanics without pressure
    \item Practicing character strategies
    \item Unlocking secret characters and achievements
    \item Solo entertainment
\end{itemize}

\textbf{Features:}
\begin{itemize}
    \item 3–4 AI opponents with adjustable difficulty (Easy, Normal, Hard)
    \item Progression system with unlockable rewards
    \item Best of 3, 5, or 7 match format
    \item Local match history and statistics
\end{itemize}

\subsection{Local Multiplayer}

Local Multiplayer enables 2–4 players to compete on the same machine using shared keyboards, mice, or gamepads. Ideal for:
\begin{itemize}
    \item Classroom tournaments
    \item Campus game hall competitions
    \item Couch gaming sessions
    \item Party events
\end{itemize}

\textbf{Features:}
\begin{itemize}
    \item Split-screen or windowed display modes
    \item Full character and power-up availability
    \item Remappable controls for each player
    \item Real-time scoreboard and HUD
    \item Best-of-series match support
\end{itemize}

\subsection{LAN Multiplayer}

LAN Multiplayer connects multiple machines over a Local Area Network (campus network, office network, or home router). Ideal for:
\begin{itemize}
    \item Campus tournaments and events
    \item LAN parties
    \item Classroom competitions
    \item Small-scale social gaming
\end{itemize}

\textbf{Technical Details:}
\begin{itemize}
    \item Host discovers clients automatically via UDP broadcast
    \item No internet connection required
    \item Latency: Typically 1–50 ms on a stable LAN
    \item Supports up to 4 players per match (expandable with multiple match instances)
    \item Persistent lobby chat and team management
\end{itemize}

\subsection{Online Multiplayer}

Online Multiplayer connects players over the internet via our dedicated VPS infrastructure. Ideal for:
\begin{itemize}
    \item Global player base
    \item Ranked competitive matches
    \item Asynchronous matchmaking (queue system)
    \item Persistent player accounts and statistics
\end{itemize}

\textbf{Technical Details:}
\begin{itemize}
    \item Authoritative server architecture (cheat-proof)
    \item Server-side snapshot replication for smooth client-side interpolation
    \item Automatic match discovery and lobby creation
    \item Latency tolerance: Playable up to 150 ms ping
    \item Player account system with rating (RR), win/loss tracking, and leaderboards
    \item Support for 100–500 concurrent players (scales with server resources)
\end{itemize}

\section{Conclusion}

Grid Survival represents a successful application of modern game development principles, including networking architecture, real-time physics, multiplayer synchronization, and user experience design. With the requested server infrastructure and on-campus installation, Grid Survival will transition from a campus project to a playable, distributable product capable of supporting a growing player base.
